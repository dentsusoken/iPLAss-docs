== 操作説明

=== 画面構成

// TODO: 画面構成のスクリーンショット

EntityListingの画面は、左側のメニューエリアと右側の検索条件・結果エリアで構成されています。

[cols="1,5a", options="header"]
|===
|エリア|説明
|タイトル|EntityListing定義の表示名またはデフォルトタイトルが表示されます。
|エンティティ・プロパティ選択メニュー|参照可能なエンティティとプロパティをツリー表示します。 +
デスクトップでは左サイドに固定表示され、モバイルではダイアログとして表示されます。
|抽出項目設定|検索結果に表示する項目を設定します。ドラッグ＆ドロップで追加・並び替えが可能。
|フィルター条件|検索条件を設定します。ドラッグ＆ドロップでフィルター項目を追加でき、条件式による複雑な条件指定が可能。
|検索結果テーブル|検索結果をテーブル形式で表示します。ソート、ページング、参照画面への遷移が可能。
|===

=== 画面操作

==== エンティティ選択

.エンティティ選択ドロップダウン
エンティティをドロップダウンから選択します。選択するとプロパティメニューにそのエンティティのプロパティが表示されます。

NOTE: 選択可能なエンティティやプロパティはEntityListingメタデータで指定可能です。未設定の場合はエンティティ権限の設定により参照可能なエンティティやプロパティが表示されます。

==== プロパティ選択

プロパティは、抽出項目またはフィルター条件に追加するための2つの方法があります。

.ダブルクリックによる追加
プロパティをダブルクリックすることで、抽出項目またはフィルター条件に直接追加できます。

.ドラッグ＆ドロップによる追加
プロパティを掴んでドラッグし、目的のエリアにドロップすることで追加できます。

* 抽出項目に追加: 抽出項目設定エリアにドラッグ＆ドロップ
* フィルター条件に追加: フィルター条件下部のドロップエリアにドラッグ＆ドロップ

==== 抽出項目設定

// TODO: 抽出項目設定のスクリーンショット

抽出項目には、以下の設定可能です。

.列の並び替え
抽出項目の列の順番はドラッグ＆ドロップで並び替え可能です。

.集計関数設定
抽出項目に対して、集計関数を指定可能です。
利用可能な集計関数はデータ型によって異なります。

[cols="1,3a", options="header"]
|===
|データ型|利用可能な集計関数
|文字列|COUNT, LISTAGG
|数値|COUNT, SUM, AVG, MIN, MAX, STDDEV_POP, STDDEV_SAMP, VAR_POP, VAR_SAMP
|日付|COUNT, MIN, MAX
|===

.表示設定
検索結果一覧の表示設定を行うことができます。

* 配置設定 +
左寄せ、中央寄せ、右寄せを設定します。未指定の場合、「左寄せ」になります。

* 列幅設定 +
検索結果の表示列幅を設定します。未指定の場合、150pxになります。

* 数値フォーマット設定（集計項目、数値項目のみ） +
数値をフォーマットするかを設定します。 +
デフォルトでは、MdcConfigServiceの「formatNumberWithComma」がtrueの場合にのみフォーマット表示されます。

==== フィルター条件

// TODO: フィルター条件設定のスクリーンショット

フィルター条件は、データ型に応じて入力方法が異なります。

[cols="1,3a", options="header"]
|===
|データ型|入力方法
|文字列|前方一致、後方一致、部分一致、完全一致、空白、空白以外
|数値|等しい、等しくない、より小さい、より大きい、範囲内
|日付|等しい、等しくない、前、後、範囲内、相対範囲
|Select|等しい、等しくない、含む、含まない
|Reference|選択、空白、空白以外
|Boolean|true、false
|===

===== 条件式

フィルターに条件が1つでも追加されると、条件式の入力エリアが表示されます。
この条件式には各フィルターの組み合わせ条件を指定可能です。

`${フィルタの行番号}` で対象の条件を指定します。

NOTE: 条件式には、カッコ、AND、OR、NOT を利用可能です。

フィルター条件式が未指定の場合は、同一プロパティはOR条件として、他プロパティに対してはAND条件として検索します。

.例：条件式未指定の場合
----
( 販売ステータス = '準備中' OR 販売ステータス = '販売中' ) AND ( 在庫数 < 500 OR 在庫数 < 100 )
----

.例：条件式指定 `($1 and $2) or ($3 and $4)` の場合
----
( 販売ステータス = '準備中' AND 在庫数 < 500 ) OR ( 販売ステータス = '販売中' AND 在庫数 < 100 )
----

===== 相対範囲（Relative Range）

日付型のフィルターでは、相対範囲指定が可能です。
例えば、「今日」「今週」「今月」「3ヶ月前」等の相対的な期間指定ができます。

設定可能な相対範囲は、EntityListing定義の`relativeRanges`プロパティで指定可能です。

===== Having条件（集計後の絞り込み）

抽出項目に集計関数がある場合、検索結果に対してさらに絞り込み（Having条件）を行うことができます。

.絞り込み条件の編集方法

ダイアログモード（デフォルト）::
絞り込みボタンをクリックすると、専用ダイアログが表示されます。

直接編集モード::
フィルターの前に絞り込み対象を指定する領域が表示されます。

==== 検索実行

.検索ボタン
条件を設定後、検索ボタンをクリックすると検索が実行されます。

.ページネーション
検索結果テーブル下部のページネーションで、表示するページを選択できます。
1ページあたりの表示件数はデフォルトで10件です。

==== 検索結果一覧

===== ソート

列ヘッダーをクリックことで、その列でのソート（昇順・降順）が可能です。

===== OID・Nameプロパティのリンク表示

抽出項目にオブジェクトID（OID）が含まれている場合、リンクとして表示されます。
リンクをクリックすることで、エンティティの参照ダイアログが開きます。

NOTE: OIDプロパティをカスタマイズしている場合（複合指定）は、リンク表示されません。

===== Referenceプロパティのリンク表示

抽出項目にReferenceプロパティが含まれている場合、リンクとして表示されます。
デフォルトでは`default`Viewで`編集可`として参照画面が開きます。

メタデータを設定することで、参照時のView指定や編集可能とするかの設定が可能です。

==== ファイルダウンロード

検索結果をCSVまたはEXCELファイルとしてダウンロードできます。

.ダウンロード手順
. ダウンロードボタンをクリック
. ファイル形式（CSV/EXCEL）を選択
. 文字コード（CSVの場合）を選択
. ダウンロードをクリック

NOTE: ファイルダウンロードは、EntityListing定義で`canFileDownload`がtrueに設定されている場合のみ利用可能です。
